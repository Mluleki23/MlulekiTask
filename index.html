Comparing Windows, macOS, and Linux

Introduction

This blog post compares the three popular operating systems: Windows, macOS, and Linux.

Files and Directories

File Organization

  • Windows: hierarchical structure, uses backslash (\) for file paths
  • macOS: hierarchical structure, uses colon (:) or slash (/) for file paths
  • Linux: hierarchical structure, uses slash (/) for file paths

Naming Conventions

  • Case sensitivity: Windows (not case-sensitive), macOS (case-insensitive but preserves case), Linux (case-sensitive)
  • Naming restrictions: varying restrictions on characters and length

File Systems

  • Windows: NTFS, FAT32 (used in USB drives, etc.)
  • macOS: APFS, HFS+
  • Linux: ext4, ext3, FAT32 (used in specific cases)

Executable File Extensions

Windows

  • .exe, .bat, .cmd
  • Script execution: PowerShell, Command Prompt

macOS

  • .app, .command, .tool
  • Script execution: Terminal, shell access

Linux

  • No specific extension, executable permissions required
  • Script execution: Terminal, shell access (bash, zsh, etc.)
